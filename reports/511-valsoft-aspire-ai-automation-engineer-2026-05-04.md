## H) Draft Application Answers

**Why this role?**
> The 70/30 internal-vs-external AI agent split — replacing manual workflows inside portfolio companies and turning some into standalone products — is what I've been doing implicitly for 2.5 years. At Progress Solutions I built healthcare RAG systems with ~35% retrieval precision improvement and agentic LLM workflows with >30% hallucination reduction; I shipped multi-agent claims pipelines with schema-validated JSON contracts between agents (Intake → Validation → Consistency → Duplicate Detection → Fraud Risk Scoring), so I know what production-grade agent chains look like under HIPAA-conscious operational discipline. As a solo founder I also shipped Manga Lens to the Chrome Web Store: 4 AI vision providers (Claude / GPT-4o mini / GPT-4.1 Nano / Ollama) routed through a per-provider payload abstraction (WebP cloud / JPEG Ollama after CUDA crashes), with privacy policy, narrowed permissions, and 29 site configs — all customer-facing. Aspire's permanent-holder model + vertical SaaS portfolio + zero-to-prototype mandate looks like the cleanest place I've seen for this builder pattern to compound.

**Walk me through a zero-to-prototype build.**
> Dream Decoder. The naive direct-prompt orchestration was failing first-pass on image generation across Replicate and Perplexity Sonar APIs — alignment between user dream → poetic interpretation → 16:9 image was inconsistent and we were burning generations on misses. I introduced an intermediate structured-prompt transformation layer: extract semantic anchors, normalize tone, project into a 16:9-friendly visual scaffold. Result: ~30% alignment lift and ~25-30% first-pass image success rate improvement. The lesson I'd carry into Aspire: for an AI automation product, the prototype is the proof that the customer's downstream behavior actually works. Build the transformation layer, ship it, measure it.

(2) Variable mechanics — is "% of internal cost savings" computed on labor displaced or on dollar savings, and is "% of new-product revenue" recognized on ARR or first-paying-customer trigger?
