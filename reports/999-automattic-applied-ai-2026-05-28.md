## H) Draft Application Answers

**"Why Automattic?"**
> Automattic is one of the few places where shipping AI features actually means shipping to millions of users — and where the underlying stack (PHP, TS, React) is the same stack I built my first marketplace on. I want to take the production-LLM discipline I've built in healthcare (RAG retrieval precision, agentic eval harnesses, audit trails) and apply it across WordPress.com / WooCommerce / Jetpack where the surface area for AI is enormous and the audience is global.

**"Most impactful AI feature you've shipped"**
> Manga Lens — a Chrome Web Store extension that translates manga and webtoon panels in real time across 29 sites. The interesting technical bit isn't the translation itself, it's the 4-provider abstraction (Claude Sonnet, GPT-4o mini, GPT-4.1 Nano, Ollama local) with per-provider payload handling. Without that boundary, a single provider's rate limit or CUDA crash on JPEG breaks the user experience. With it, the consumer side is decoupled from provider variance — which is exactly the kind of discipline I'd want to bring to WP's AI feature surface.

**"Open source / public work"**
> Manga Lens is published to Chrome Web Store with privacy policy and narrowed host permissions. Portfolio (Dream Decoder, Pixel Character Synthesis Engine, Claims Fraud Risk Pipeline) is open on GitHub. Happy to ramp into Automattic's WP plugin contribution flow.
