## F — Interview Prep (STAR+R)

**STAR 1 — SDK integration depth (relevant to WorkOS POC/onboarding function):**
- **Situation:** The Manga Lens Chrome extension needed to support multiple AI providers (OpenAI, Anthropic, Google) with a single interface — users could switch providers without changing their workflow.
- **Task:** Design and implement a provider-agnostic API integration layer that handled authentication, rate limiting, and error handling per provider.
- **Action:** Built an abstraction layer in TypeScript that normalized each provider's REST API into a common interface; managed API key storage in Chrome's secure storage; wrote provider-specific error handlers so the UX remained consistent regardless of which backend was called.
- **Result:** Extension shipped to the Chrome Web Store; multi-provider support is cited as the primary differentiator in user reviews.
- **Reflection:** This pattern — abstract the provider, expose a consistent interface to the consumer — is exactly what Solutions Engineers do when helping developers integrate WorkOS across languages. The lesson was to design for the developer's workflow first, not the API's structure.

**STAR 2 — Rapid API onboarding under delivery pressure (relevant to POC velocity):**
- **Situation:** The Agentic Healthcare Claims Pipeline required integrating a new LLM orchestration layer (LlamaIndex) into an existing Python/FastAPI codebase with a tight project timeline.
- **Task:** Understand LlamaIndex's indexing and retrieval APIs, integrate them without disrupting existing endpoints, and validate performance improvement on real claims data.
- **Action:** Read the LlamaIndex docs systematically, built a sandboxed integration branch, ran A/B comparisons against the prior retrieval method, then merged only after confirming the ~35% RAG precision improvement on held-out test cases.
- **Result:** Integration shipped on schedule; precision improved ~35%; the approach of sandboxed POC → validated comparison → production merge became the team's standard for new framework adoption.
- **Reflection:** POC velocity with rigor is a differentiator for SE roles — customers trust SEs who can build a working integration quickly *and* explain why it performs the way it does.
