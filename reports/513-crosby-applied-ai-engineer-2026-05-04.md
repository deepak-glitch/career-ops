## H) Draft Application Answers

**Why does this role excite you?**
> Crosby's framing — "the right way to deploy AI in law isn't by selling software, it's by selling outcomes" — maps directly to how I've been building. At Progress Solutions I built healthcare RAG systems with ~35% retrieval precision improvement and >90% retrieval-grounded response alignment, plus a multi-agent claims pipeline (Intake → Validation → Consistency → Duplicate Detection → Fraud Risk Scoring) with schema-validated JSON contracts between agents to prevent cascading hallucinations — that's the same complex-RAG + agentic-workflows + production-quality stack you describe, just in a HIPAA-conscious regulated domain instead of legal review. As a solo founder I shipped Manga Lens to the Chrome Web Store with 4 AI vision providers (Claude Sonnet, GPT-4o mini, GPT-4.1 Nano, Ollama) routed through a per-provider payload abstraction (WebP cloud / JPEG Ollama after CUDA crashes), proving I can own a customer-visible feature end-to-end. Crosby's outcomes-over-software model is exactly the wedge I want to push into next.

**Walk me through a complex RAG + agentic system you built.**
> Healthcare claims fraud risk pipeline at Progress Solutions. The Stage was: claims processing bottlenecked by manual fraud review across multi-step intake → validation → duplicate detection → risk scoring, with cascading hallucination risk in naive multi-agent setups. Task: build a production-ready agentic pipeline that wouldn't fall over under real claim volumes. Action: designed a 5-agent chain (Intake Normalization → Validation → Consistency Analysis → Duplicate Detection → Fraud Risk Scoring) with schema-validated JSON contracts between every adjacent pair of agents (so a downstream agent that sees malformed upstream output fails loud, not silent), plus RAG-grounded CPT/ICD validation against a vector-indexed policy corpus, plus ANN similarity search for duplicate-claim detection, plus explainable risk scoring with audit-ready reasoning traces for compliance. Result: production multi-agent system with cascading-hallucination prevention. Reflection: the schema contracts are what separates agentic-prototype from agentic-production — and that's exactly the discipline Crosby's legal-review pipeline will need.

**Selective fine-tuning — when have you decided to fine-tune vs prompt-engineer?**
> Pixel Character Synthesis. Off-the-shelf Stable Diffusion could not hold identity consistency for sprite-sheet generation across pose variations — prompt engineering hit a ceiling at frame 3-4 of an animation. I fine-tuned with LoRA for identity consistency, added ControlNet + OpenPose/MediaPipe for pose conditioning, and used latent-space consistency for animation smoothness, with an LLM agent orchestrator decomposing high-level prompts into generation tasks. The lesson: selective fine-tuning is a precision tool, not a default — apply only when off-the-shelf falls short, which is exactly your framing.
