## H) Draft Application Answers (score 3.8 — included for ready-to-apply)

**Why are you interested in Novoflow?**
> Novoflow is solving the exact problem I've been working on in healthcare AI for the last 2.5 years — making AI agents work reliably inside real medical operations. At Progress Solutions I've built a healthcare RAG system with ~35% retrieval precision gain, agentic LLM workflows for eligibility and care navigation with >30% hallucination reduction, a multi-agent claims processing pipeline with schema-validated JSON contracts, and HIPAA-conscious governance + stakeholder limitation docs that actually unlocked clinician adoption. I want to do that work for clinics directly — get into the EHR, debug live, write the Python and TS, and own the customer outcome.

**Tell us about an AI system you've debugged inside a customer environment.**
> The healthcare RAG at Progress Solutions started drifting on retrieval precision week-over-week — clinicians flagged it. I recreated the affected query slices, ran the eval harness against current vs prior week, and isolated the regression to the chunking layer (a recent ingestion changed chunk size distributions). I retuned the chunker thresholds, re-ran eval, and pushed the fix. The deeper lesson was that the real customer-environment problem wasn't the chunking — it was the absence of an eval harness alarm that would have flagged the drift before clinicians did. So I added drift signals into the audit trail and gave the clinicians a dashboard. After that, we caught issues before they did.
