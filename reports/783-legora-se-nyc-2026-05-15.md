## F — Interview Prep (STAR+R)

**STAR 1 — LLM precision in a regulated domain (directly maps to legal AI needs):**
- **Situation:** The Agentic Healthcare Claims Pipeline needed to classify and route insurance claims using RAG over unstructured medical records — a domain where LLM errors have direct financial and compliance consequences.
- **Task:** Improve retrieval precision to the point where the system could be trusted in a production claims review workflow, reducing manual review load without increasing error rate.
- **Action:** Redesigned the chunking and embedding strategy, added a re-ranking step using cross-encoder models, and implemented a confidence threshold below which claims were routed to human review rather than auto-classified. Ran systematic evaluation across held-out test sets.
- **Result:** ~35% improvement in RAG precision; >30% reduction in hallucination rate; 15–20% improvement in high-risk claim recall. The system moved from prototype to a production-grade triage tool.
- **Reflection:** The key insight for legal AI customers: precision is non-negotiable when the output has professional or financial consequences. The same evaluation framework — confidence thresholds, human-in-the-loop fallback, systematic precision testing — applies directly to legal document review.

**STAR 2 — Onboarding into a new domain quickly (maps to SE ramp and customer success):**
- **Situation:** Building the Dream Decoder application required integrating a FastAPI backend with a React/TypeScript frontend and deploying to production without prior experience in that specific stack combination.
- **Task:** Design and ship a full-stack application with a clean API contract between frontend and backend, meeting a self-imposed delivery deadline.
- **Action:** Studied FastAPI's async patterns and OpenAPI schema generation, designed the API contract first (before writing either the backend or frontend), then implemented both sides against the contract. Used TypeScript's type system to catch API mismatches at compile time rather than runtime.
- **Result:** Application shipped on schedule with a clean integration and zero post-launch API contract bugs.
- **Reflection:** The ability to onboard into a new stack or domain quickly — by studying the contract first, then implementing — is the same skill an SE uses when learning a customer's existing environment before integrating a new platform. Domain knowledge is learnable; the methodology for learning it fast is the differentiator.
